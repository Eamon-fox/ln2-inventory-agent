# SnowFox GUI 美化建议（仅前端视觉与交互）

> 目标：不改底层逻辑/后端，仅提升界面观感与操作体验。

## 一、当前状态（2026-02）

### 已完成

- 三栏布局保留，分割条可见性增强（拖拽更易发现）。
- Plan 入队失败可视化：错误不再只在左下角状态栏，而是出现在操作区下方的就地提示。
- Plan 阶段改为预检拦截：缺字段、位置冲突在“添加到计划”时即提示。
- 概览顶部冗余信息（容量 / 7天操作数）已移除，减少视觉噪声。
- 清理不兼容样式项（如 `cursor`），减少 QSS 解析告警。

### 不采用（删除）

- 启动画面（Splash）：当前工具型产品强调“秒开可操作”，不额外增加启动打断。

### 待优化

- 统一卡片边界密度（Overview / Operations / AI 三栏边框与内边距仍略有不一致）。
- 列表与表头层级统一（部分表格行高、列宽策略仍不完全一致）。
- 深浅主题对比度抽检（重点看 warning/error 与二级文本）。
- 中英文混排对齐（长中文字段在表格与按钮中的截断策略需要统一）。

## 二、后续实施清单（按优先级）

1. 表格系统统一
   - 统一行高、hover、选中态、表头字号和间距。
   - 统一列宽策略（Stretch vs ResizeToContents 的使用边界）。

2. 卡片与信息层级
   - 操作区、结果区、AI 区统一卡片内边距与圆角。
   - 减少重复边框，突出当前上下文（当前操作、当前预览、当前报错）。

3. 交互反馈补齐
   - Overview 刷新/耗时操作显示明确的加载状态文案。
   - 空状态继续保持“可行动引导”（下一步按钮/提示语可直接操作）。

4. 主题一致性
   - 抽检 light/dark 下关键按钮语义色（primary/danger/warning）。
   - 校验中文环境下字号、截断、省略号和 tooltip 一致性。

## 三、验收标准（UI 侧）

- 首屏 3 秒内可识别主要操作入口。
- “先查后取”路径操作无视觉歧义。
- light/dark 主题均可读、可辨、可快速定位。
- 不改变业务逻辑与数据行为。
