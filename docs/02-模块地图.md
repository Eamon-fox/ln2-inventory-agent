# 模块地图

本文档不是目录介绍，而是并行开发地图。每个模块都必须回答 7 件事：负责什么、拥有哪些路径、有哪些稳定入口、允许依赖谁、禁止依赖谁、是否适合并行、最小测试责任是什么。

本页同样采用三档模型：

- 硬约束：模块 ownership、共享路径、文档入口
- 软约束：并行策略、推荐测试集、当前主要协作面
- 背景说明：为什么这么切模块、常见失控点

## 模块总表

| 模块 ID | 主要职责 | 拥有路径 | 模块文档 |
| --- | --- | --- | --- |
| `gui_presentation` | Qt 界面、面板、对话框、表格、视觉交互 | `app_gui/ui/` | `modules/10-界面展示层.md` |
| `gui_application` | GUI 流程编排、事件总线、计划执行、主窗口协作 | `app_gui/application/`、`app_gui/plan_*.py` | `modules/11-界面应用层.md` |
| `agent_runtime` | ReAct 循环、工具调度、工具 hooks、输入校验 | `agent/` | `modules/12-智能体运行时.md` |
| `inventory_core` | Tool API、工具注册、库存规则、校验、YAML 业务处理 | `lib/`，排除导入迁移专题路径 | `modules/13-库存核心.md` |
| `migration_import` | 导入流程、迁移策略、迁移模板、验收 | `migrate/`、`migration_assets/`、`lib/import_*`、`lib/migrate_*`、`app_gui/import_journey.py` | `modules/14-导入迁移.md` |
| `release_packaging` | 打包、安装、版本同步、发布脚本 | `installer/`、`scripts/`、`*.spec`、`latest.json`、`CHANGELOG.md` | `modules/15-发布与安装.md` |

接到任务后，先用本页确定模块归属，再去读对应模块文档，不要直接跳进具体代码文件。

## `gui_presentation`

### 负责范围

- 纯界面组件
- 面板布局、表格渲染、对话框交互
- 面板内局部交互状态

### 拥有路径

- `app_gui/ui/`

### 稳定入口

跨模块调用 `OperationsPanel` 时，只允许依赖以下稳定公共方法：

- `apply_meta_update`
- `set_migration_mode_enabled`
- `update_records_cache`
- `set_prefill`
- `set_prefill_background`
- `set_add_prefill`
- `set_add_prefill_background`
- `add_plan_items`
- `execute_plan`
- `clear_plan`
- `reset_for_dataset_switch`
- `on_export_inventory_csv`
- `emit_external_operation_event`
- `print_plan`
- `print_last_executed`
- `on_undo_last`
- `remove_selected_plan_items`
- `on_plan_table_context_menu`
- `refresh_plan_store_view`
- `OverviewPanel.bind_plan_store`
- `OverviewPanel.refresh_plan_store_view`
- `AIPanel.apply_runtime_settings`
- `AIPanel.prepare_external_prompt`
- `AIPanel.runtime_settings_snapshot`
- `AIPanel.has_running_task`

### 允许依赖

- `gui_application`
- `inventory_core` 的只读格式化或展示辅助能力

### 禁止依赖

- `lib.tool_api_write*`
- `lib.tool_api_write_validation`
- `agent/`

### 并行说明

- 适合并行，但只限不同面板或不同 UI 子组件。
- 一旦需要改 `app_gui/main.py`、`app_gui/main_window_flows.py` 或 `app_gui/tool_bridge.py`，就不再是纯 `gui_presentation` 任务。

### 最小测试责任

- `pytest -q tests/integration/gui`

## `gui_application`

### 负责范围

- GUI 流程编排
- 事件总线
- 计划执行与结果整理
- 主窗口级协作与状态同步
- GUI 到核心能力的桥接
- 本地开放 API 的 transport、allowlist、只读查询/校验 route、受管数据集会话切换与 GUI handoff

### 拥有路径

- `app_gui/application/`
- `app_gui/plan_*.py`

### 共享入口但非独占路径

以下文件由本模块主导语义，但因为跨模块高度共享，视为共享瓶颈点：

- `app_gui/main.py`
- `app_gui/main_window_flows.py`
- `app_gui/tool_bridge.py`

### 稳定入口

- `DatasetUseCase.switch_dataset`
- `MigrationModeUseCase.set_mode`
- `PlanExecutionUseCase.report_operation_completed`
- `PlanRunUseCase.execute`
- `DatasetLifecycleUseCase.resolve_startup_yaml_path` / `create_dataset` / `rename_dataset` / `delete_dataset`
- `SettingsDialogSubmission`

### 允许依赖

- `inventory_core`
- `migration_import`
- `gui_presentation`

### 禁止依赖

- 直接依赖具体 UI 私有实现细节
- 在应用层复制库存规则

### 并行说明

- 可以和 `gui_presentation` 并行，但前提是不同时修改共享瓶颈点。
- 若任务涉及主窗口装配、跨面板 wiring、桥接方法签名，默认串行。

### 最小测试责任

- `pytest -q tests/integration/plan tests/integration/gui`

## `agent_runtime`

### 负责范围

- ReAct 循环
- LLM 调用
- 工具调度
- 工具 hooks
- 工具输入校验与恢复提示

### 拥有路径

- `agent/`

### 稳定入口

- `ReactAgent`
- `AgentToolRunner.list_tools`
- `AgentToolRunner.run`
- `AgentToolRunner._dispatch_handlers` 生成的调度面

### 允许依赖

- `inventory_core`
- `migration_import` 提供的迁移工具与模板能力

### 禁止依赖

- `PySide6`
- 复制 `inventory_core` 的业务规则
- 直接写 YAML 而绕过统一 Tool API

### 并行说明

- 与 `gui_presentation` 一般可以并行。
- 若任务涉及工具注册、统一契约或 GUI 侧桥接方法，同样会命中共享瓶颈点。

### 最小测试责任

- `pytest -q tests/integration/agent tests/unit/test_tool_hooks.py`

## `inventory_core`

### 负责范围

- 统一 Tool API
- 工具注册与契约导出
- 库存读写规则
- 校验逻辑
- YAML 业务处理
- 自定义字段与通用业务服务

### 拥有路径

- `lib/`
- 但不包含 `lib/import_*` 与 `lib/migrate_*`

### 稳定入口

- `lib/tool_api.py`
- `lib/tool_registry.py`
- `lib/domain/commands.py`
- `lib/domain/events.py`

### 允许依赖

- 基础设施能力只能向下被调用，不能反向引入 GUI/Agent

### 禁止依赖

- `PySide6`
- GUI 状态对象
- GUI 组件或窗口类

### 并行说明

- 大部分核心改动不适合多人同时碰统一入口。
- 若能拆成“规则实现”与“统一入口接线”两段，可内部并行。

### 最小测试责任

- `pytest -q tests/integration/inventory tests/unit`

## `migration_import`

### 负责范围

- 导入流程
- 迁移策略
- 导入验收
- 迁移模板与迁移工作区

### 拥有路径

- `migrate/`
- `migration_assets/`
- `lib/import_*`
- `lib/migrate_*`
- `app_gui/import_journey.py`

### 稳定入口

- `app_gui/import_journey.py`
- `lib/import_acceptance.py`
- `lib/migrate_cell_line_policy.py`

### 允许依赖

- `inventory_core`
- `gui_application`

### 禁止依赖

- 直接把迁移细节塞进 `gui_presentation`
- 在迁移模块里定义独立库存真相源

### 并行说明

- 可与 `gui_presentation` 并行。
- 只要涉及导入后切换数据集、主窗口 handoff 或工具注册，就要与 `gui_application` 或 `inventory_core` 协调。

### 最小测试责任

- `pytest -q tests/integration/migration`

## `release_packaging`

### 负责范围

- 安装与打包
- 发布脚本
- 版本号同步
- 安装器与发布资产

### 拥有路径

- `installer/`
- `scripts/`
- `ln2_inventory.spec`
- `ln2_inventory.mac.spec`
- `latest.json`
- `CHANGELOG.md`

### 稳定入口

- `installer/windows/`
- `installer/mac/`
- `scripts/README.md`

### 允许依赖

- 读取版本文件与构建产物信息

### 禁止依赖

- 把业务规则、库存校验或导入策略写进打包逻辑

### 并行说明

- 大多可以与业务模块并行。
- 若需要改版本号、更新文案或网站同步脚本，注意与翻译和发布节奏协调。

### 最小测试责任

- 至少执行与改动直接相关的构建或脚本自检

## 模块间并行默认策略

### 默认可并行

- `gui_presentation` 与 `agent_runtime`
- `gui_presentation` 与 `migration_import`
- `release_packaging` 与其他大多数业务模块

### 默认谨慎并行

- `gui_application` 与 `gui_presentation`
- `agent_runtime` 与 `inventory_core`
- `migration_import` 与 `inventory_core`

### 默认串行

- 任意模块与共享瓶颈点改动同时发生时
- 多个模块同时修改 `lib/tool_api.py`
- 多个模块同时修改 `lib/tool_registry.py`
- 多个模块同时修改 `app_gui/main.py`

## 任务切分模板

当一个需求跨多个模块时，优先按下面顺序拆：

1. 共享入口或契约变更
2. 模块内部实现变更
3. UI 接线与文案合并
4. 测试与回归收尾

不要反过来从 UI 直接拉动底层大改，这会让 agent 很快失去上下文边界。
